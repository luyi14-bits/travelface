TravelFace

基于人脸画像与情绪识别的个性化旅游推荐系统 · 技术文档
纯 NumPy 视觉管线 · 零外部依赖 · Streamlit 全栈

一、项目概述

1.1 项目背景

随着人工智能技术的快速发展，个性化推荐系统已广泛应用于电商、音乐、视频等领域。然而在旅游行业，传统的推荐方式仍主要依赖用户的历史浏览记录和评分数据，未能充分利用用户的即时情绪状态和视觉肖像信息。本项目创新性地将计算机视觉人脸检测、用户画像分析、情绪识别与 AIGC 智能体相结合，构建了一套"拍张照片即可获得专属旅行路线"的智能推荐系统。

1.2 核心功能

功能模块描述
人脸检测基于纯 NumPy 的 YCbCr 肤色检测算法，零外部依赖，精准定位照片中人脸位置与数量
用户画像基于图像特征分析（亮度梯度的年龄段估计，RGB 通道比的性别推测），构建用户基础画像
情绪识别基于面部亮度和对比度特征的七种情绪分类（开心/悲伤/愤怒/惊讶/恐惧/厌恶/中性）
旅游推荐集成 OpenAI 兼容大语言模型 API，根据画像+情绪生成 3 条个性化路线
优雅降级所有视觉分析模块均内置兜底机制，不可用时自动降级为估测值

1.3 技术亮点

  • 零外部视觉依赖：人脸检测/用户画像/情绪识别均基于纯 NumPy + Pillow，无 OpenCV / MediaPipe / DeepFace / TensorFlow 依赖
  • 极速启动：首次运行即开即用，无模型下载等待，总安装体积 < 50MB
  • 双模式推荐：支持接入 OpenAI 兼容 API，未配置 API Key 时自动切换到内置模拟推荐
  • 双主题 UI：暗色/浅色模式自动适配，按钮/Metric/卡片均带悬停放大发光阴影动效

四、模块详解

4.1 人脸检测模块 (FaceDetector)

算法：YCbCr 色彩空间肤色检测 + Flood Fill 连通域分析

将 RGB 转换到 YCbCr 色彩空间，肤色在 Cb-Cr 平面具有稳定的聚类区间（Cb 77~127, Cr 133~173），该区间由学术界多年肤色建模确定，对不同人种和光照具有较好鲁棒性。

处理流程：

RGB 图像 → float32 → Cb/Cr 颜色空间转换 → 肤色掩膜 = (77≤Cb≤127) & (133≤Cr≤173) → Flood Fill 连通域提取（步长2加速扫描） → 六层筛选 → 取综合打分最高唯一区域 → 12% padding → 返回 [{box, crop, detector}]

六层筛选校验：

校验层阈值作用
尺寸占图片 0.8%~35%排除过小噪点和过大背景
像素数≥ 200排除零碎肤色散点
纵横比0.6~1.7人脸近似正方形，排除长条形区域
填充率≥ 35%人脸是实心的，排除散点状区域
位置中心在画面 5%~60% 高度人脸不可能在画面极底部
综合打分填充率×0.5 + 位置×0.3 + 面积×0.2取最优唯一区域

降级机制：肤色像素占比 < 1.2% 或所有区域未通过筛选时，自动降级为画面中心偏上的智能猜测（30% × 35%）。

4.2 用户画像模块 (UserProfiler)

算法：基于图像特征的启发式分析

年龄估计（基于面部区域平均亮度）：

  • brightness < 100 → 中老年（较暗肤色/皱纹吸光）
  • brightness > 180 → 少年（年轻肌肤反光强）
  • 100 ≤ brightness ≤ 180 → 青年

性别推测（基于 R/(G+B) 皮肤红润度）：

  • R/(G+B) > 1.1 → 女性
  • R/(G+B) ≤ 1.1 → 男性

4.3 情绪识别模块 (EmotionRecognizer)

算法：基于面部亮度和对比度的启发式分类

条件判定情绪
亮度 > 180 且 对比度 > 40开心（笑容使面部更亮、纹路增多）
亮度 < 120悲伤（低沉表情使面部偏暗）
对比度 < 25中性（面部肌肉放松、纹理均匀）
其他中性（兜底）

双模式设计：模块同时支持 DeepFace 调用（若本地已安装），DeepFace 不可用时自动降级为上述启发式算法。

七、开发问题与解决方案

问题1：OpenCV 在 Python 3.14 上编译失败

现象：pip install opencv-python-headless → fatal error: Python.h: No such file or directory

解决：完全移除 OpenCV，用 Pillow 替代所有图像处理操作。

问题2：MediaPipe 隐式依赖 cv2 导致连锁失败

解决：移除 MediaPipe，人脸检测改用纯 NumPy YCbCr 肤色检测算法。

问题3：DeepFace + TensorFlow 安装体积过大

解决：移除两个包，用户画像和情绪识别改用NumPy 图像特征启发式算法。

问题4：浅色模式下文字完全不可见

解决：为所有暗色模式样式添加 [data-theme="light"] 浅色覆盖规则。

问题5：肤色检测误判多人脸（迭代记录）

第一阶段：墙壁/衣服误判 → 添加 RGB 双重校验

第二阶段：RGB 过严导致漏检 → 移除 RGB 约束，恢复经典 YCbCr

第三阶段：单人被判定为 3~4 人 → 改为"只取综合打分最高 1 个区域"

最终方案：宽松 YCbCr + 六层轻量筛选，取唯一最优结果

问题6：部署版缺少悬停动效

解决：恢复完整悬停动效系统，暗色/浅色独立适配。
