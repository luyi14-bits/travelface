# TravelFace · 基于人脸画像与情绪识别的个性化旅游推荐系统

## 一、项目概述

### 1.1 项目背景

随着人工智能技术的快速发展，个性化推荐系统已广泛应用于电商、音乐、视频等领域。然而在旅游行业，传统的推荐方式仍主要依赖用户的历史浏览记录和评分数据，未能充分利用用户的即时情绪状态和视觉肖像信息。本项目创新性地将计算机视觉人脸检测、用户画像分析、情绪识别与 AIGC 智能体相结合，构建了一套"拍张照片即可获得专属旅行路线"的智能推荐系统。

### 1.2 核心功能

| 功能模块 | 描述 |
|---------|------|
| **人脸检测** | 基于纯 NumPy 的 YCbCr 肤色检测算法，零外部依赖，精准定位照片中人脸位置与数量 |
| **用户画像** | 基于图像特征分析（亮度梯度的年龄段估计，RGB 通道比的性别推测），构建用户基础画像 |
| **情绪识别** | 基于面部亮度和对比度特征的七种情绪分类（开心、悲伤、愤怒、惊讶、恐惧、厌恶、中性） |
| **旅游推荐** | 集成 OpenAI 兼容大语言模型 API，根据画像+情绪生成 3 条个性化路线 |
| **优雅降级** | 所有视觉分析模块均内置兜底机制，情绪识别不可用时自动降级为估测值 |

### 1.3 技术亮点

- **零外部视觉依赖**：人脸检测、用户画像、情绪识别三大模块均基于纯 NumPy + Pillow 实现，无需下载任何预训练模型权重，无 OpenCV / MediaPipe / DeepFace / TensorFlow 依赖
- **极速启动**：首次运行即开即用，无模型下载等待，总安装体积 < 50MB
- **双模式推荐**：支持接入 OpenAI 兼容 API 进行智能推荐，未配置 API Key 时自动切换到内置模拟推荐数据
- **双主题 UI**：暗色/浅色模式自动适配，暗色模式为深蓝紫科技风渐变背景 + 玻璃拟态卡片，浅色模式为淡蓝灰柔和渐变 + 高对比度深色文字

## 四、模块详解

### 4.1 人脸检测模块 (FaceDetector)

**文件**：[src/vision/face_detector.py](file:///d:/Desktop/travelface/src/vision/face_detector.py)

**算法**：YCbCr 色彩空间肤色检测 + Flood Fill 连通域分析

**肤色检测原理**：
- 将 RGB 图像转换到 YCbCr 色彩空间
- 肤色在 Cb-Cr 平面具有稳定的聚类区间：77 ≤ Cb ≤ 127, 133 ≤ Cr ≤ 173
- 该区间由学术界多年肤色建模研究确定，对不同人种和光照具有较好鲁棒性

**处理流程**：

```
RGB 图像 → float32 转换
         → Cb = 128 - 0.169R - 0.331G + 0.5B
         → Cr = 128 + 0.5R - 0.419G - 0.081B
         → 肤色掩膜 = (77≤Cb≤127) & (133≤Cr≤173)
         → Flood Fill 连通域提取（步长2加速扫描）
         → 六层筛选校验
         → 取综合打分最高的唯一区域
         → 12% padding 扩展边界框
         → 返回 [{box, crop, detector}]
```

**六层筛选校验**：

| 校验层 | 阈值 | 作用 |
|--------|------|------|
| **尺寸** | 占图片 0.8%~35% | 排除过小噪点和过大背景 |
| **像素数** | ≥ 200 | 排除零碎肤色散点 |
| **纵横比** | 0.6~1.7 | 人脸近似正方形，排除长条形区域 |
| **填充率** | ≥ 35% | 人脸是"实心"的，排除散点状区域 |
| **位置** | 中心在画面 5%~60% 高度 | 人脸不可能在画面极底部 |
| **综合打分** | 填充率×0.5 + 位置×0.3 + 面积×0.2 | 取最优唯一区域 |

**降级机制**：当肤色像素占比 < 1.2% 或所有区域未通过筛选时，自动降级为画面中心偏上的智能猜测（取图片宽 30% × 高 35% 的居中偏上区域），确保非人脸图片也能运行。

### 4.2 用户画像模块 (UserProfiler)

**文件**：[src/vision/user_profiler.py](file:///d:/Desktop/travelface/src/vision/user_profiler.py)

**算法**：基于图像特征的启发式分析

**年龄估计**：基于面部区域平均亮度：
- `brightness < 100` → 中老年（较暗肤色/皱纹区域吸光）
- `brightness > 180` → 少年（年轻肌肤反光强）
- `100 ≤ brightness ≤ 180` → 青年

**性别推测**：基于 R 通道与 G+B 通道的比例（皮肤红润度）：
- `R/(G+B) > 1.1` → 女性（肤色更红润）
- `R/(G+B) ≤ 1.1` → 男性

**接口**：
```python
class UserProfiler:
    def analyze(self, face_crop: np.ndarray) -> dict
    # 返回: {age_group: "青年"|"中年"|... , gender: "Female"|"Male"|"未知", confidence: float}
```

### 4.3 情绪识别模块 (EmotionRecognizer)

**文件**：[src/vision/emotion_recognizer.py](file:///d:/Desktop/travelface/src/vision/emotion_recognizer.py)

**算法**：基于面部亮度和对比度的启发式分类

**识别逻辑**：

| 条件 | 判定情绪 |
|------|---------|
| 亮度 > 180 且 对比度 > 40 | 开心（笑容使面部更亮、纹路增多） |
| 亮度 < 120 | 悲伤（低沉表情使面部偏暗） |
| 对比度 < 25 | 中性（面部肌肉放松、纹理均匀） |
| 其他 | 中性（兜底） |

**接口**：
```python
class EmotionRecognizer:
    def recognize(self, face_crop: np.ndarray) -> dict
    # 返回: {emotion: "开心", emotion_en: "happy", confidence: 0.6, scores: {...}}
```

**双模式设计**：模块同时支持 DeepFace 调用（本地环境若已安装），若 DeepFace 不可用则自动降级为上述启发式算法。

### 4.4 旅游推荐模块 (TravelAgent)

**文件**：[src/llm/travel_agent.py](file:///d:/Desktop/travelface/src/llm/travel_agent.py)

**核心设计**：双模式切换

| 模式 | 触发条件 | 实现 |
|------|---------|------|
| **LLM 模式** | Streamlit Secrets 中配置了 `OPENAI_API_KEY` | 调用 OpenAI 兼容 API，System Prompt 引导输出结构化 JSON |
| **模拟模式** | 未配置 API Key | `_mock_response()` 内置 12 条路线（覆盖开心/悲伤/中性三种情绪场景） |

**LLM 调用流程**：

```
vision_result → _build_user_prompt() → 拼接 System Prompt + 用户画像

                                     → _parse_response() JSON 提取
                                     → {routes: [...]}
```

**模拟推荐覆盖**：

| 情绪 | 路线1 | 路线2 | 路线3 |
|------|-------|-------|-------|
| 负面（悲伤/愤怒/恐惧） | 海边治愈 🌊 | 山林静修 🏔️ | 艺术疗愈 🎨 |

## 五、UI 设计

### 5.1 双主题设计

采用**暗色/浅色自动适配**的设计策略：

**暗色模式**（默认）：
- 背景：`linear-gradient(135deg, #0b0b2a, #1a1a4e, #16213e, #0f3460)` 四段深蓝紫渐变
- 卡片：`rgba(255,255,255,0.06)` 半透明磨砂效果
- 文字：`#e0e0ff` / `#fff` 浅色系
- 按钮：紫蓝渐变 `linear-gradient(135deg, #7b2ff7, #00d2ff)`

**浅色模式**（通过 `[data-theme="light"]` 选择器覆盖）：
- 背景：`linear-gradient(135deg, #e8ecf8, #dce4f5, #eaf0fa, #f0f4ff)` 淡蓝灰柔和渐变
- 卡片：`rgba(0,0,0,0.04)` 浅灰半透明
- 文字：`#2a2a4e` / `#1a1a2e` 深色系
- 按钮：`linear-gradient(135deg, #6a1fe7, #0098cc)`

### 5.2 交互动效

所有可交互元素均带有 CSS `transition` 平滑过渡和悬停效果：

| 元素 | 悬停效果 | 实现 |
|------|---------|------|
| 按钮 | 放大 1.04x + 紫光阴影 | `transform: scale(1.04); box-shadow: 0 4px 20px rgba(123,47,247,0.45)` |
| Metric 卡片 | 上移 2px + 紫光阴影 | `transform: translateY(-2px); box-shadow: 0 6px 18px rgba(123,47,247,0.25)` |
| 旅行卡片 | 上移 4px + 黑影加深 | `transform: translateY(-4px); box-shadow: 0 10px 30px rgba(0,0,0,0.30)` |

暗色/浅色两种模式下的悬停阴影颜色独立适配（暗色为紫光、浅色为灰影），所有动效通过 `transition: transform 0.2s ease` 实现平滑过渡。

### 5.3 卡片封面设计

由于外网图片 API 在国内不可用，采用纯 CSS 封面方案：
- 每条路线使用不同的渐变色对（紫/红/蓝三组轮换）
- 大号场景图标（🏔️ 🏖️ 🏯）作为视觉焦点
- 白色粗体目的地名称（宽字距 + 文字阴影）

### 5.4 HTML 渲染方案

旅行卡片和用户详情卡片使用 `st.html()` 直出 HTML，完全绕过 Markdown 解析器，确保复杂 DOM 结构正确渲染。文字颜色通过 CSS class（`fd-title`/`fd-label`/`fd-emotion`/`fd-conf`）控制，利用 `[data-theme]` 选择器自动适配暗色/浅色模式。

## 七、开发问题与解决方案

### 问题1：OpenCV 在 Python 3.14 上编译失败

**现象**：`pip install opencv-python-headless` 报 `fatal error: Python.h: No such file or directory`

**原因**：Python 3.13+ 移除了 `python3.dll`，opencv-python-headless 在这些版本上无预编译 wheel，需要从源码编译但缺少 Python 开发头文件。

**解决**：完全移除 OpenCV 依赖，用 Pillow 替代所有图像处理操作（`Image.open` 代替 `cv2.imdecode`，`ImageDraw` 代替 `cv2.rectangle/putText`）。

### 问题2：MediaPipe 隐式依赖 cv2 导致连锁失败

**现象**：安装 mediapipe 成功，但 import 时 `drawing_utils.py` 内部 `import cv2` 失败。

**原因**：MediaPipe 的可视化工具模块硬依赖 OpenCV，即使只用人脸检测也会触发。

**解决**：移除 MediaPipe，用人脸检测改用纯 NumPy 实现的 YCbCr 肤色检测算法。

### 问题3：DeepFace + TensorFlow 安装体积过大

**现象**：Streamlit Cloud 部署时 `pip install deepface` 连带安装 TensorFlow（~600MB），超过免费版资源限制，安装超时或 OOM。

**解决**：移除 DeepFace 和 TensorFlow，用户画像和情绪识别改用基于 NumPy 的图像特征启发式算法。

### 问题4：浅色模式下文字完全不可见

**现象**：暗色模式 CSS 把所有文字设成白色（`color: #fff`），切换到浅色模式后白字在白背景上完全不可见。

**原因**：CSS 只定义了暗色模式样式，缺少 `[data-theme="light"]` 选择器做浅色模式覆盖。

**解决**：为所有暗色模式样式添加 `[data-theme="light"]` 对应的浅色覆盖规则，涵盖文字、卡片、Metric、按钮等所有组件。

### 问题5：整张图片被当作人脸框选

**现象**：移除 MediaPipe 后临时将整张图片当作人脸区域，框选线覆盖整张图。

**原因**：FaceDetector 被临时旁路，直接用 `box = (0, 0, w, h)` 作为人脸区域。

**解决**：实现完整的 YCbCr 肤色检测 + Flood Fill + 区域筛选的人脸定位管线，能正确框选出画面中人脸的实际位置。

### 问题6：肤色检测误判多人脸（迭代记录）

**第一阶段：墙壁/衣服误判为人脸**
- **现象**：YCbCr 肤色检测把墙壁、衣服、木头等肤色相近物体误判为人脸
- **解决**：添加 RGB 双重校验（R>95, G>40, R>G>B, 通道差>15），收紧 YCbCr 范围至 Cb 80~125 / Cr 136~170

**第二阶段：RGB 约束过于严格导致真·人脸漏检**
- **现象**：收紧后真·人脸也被过滤掉，完全检测不到
- **解决**：移除 RGB 约束，恢复经典 YCbCr 范围（77~127 / 133~173），只保留轻量过滤

**第三阶段：单人照片被判定为 3~4 人**
- **现象**：宽松肤色检测后，手上的肤色、衣服肤色区域都被当作额外人脸
- **解决**：改为"只取综合打分最高的 1 个区域"策略，不再取 Top N

**最终方案**：宽松 YCbCr + 六层轻量筛选（尺寸/像素/纵横比/填充率/位置/综合打分），取唯一最优结果。肤色检测失败时走中心猜测兜底。

### 问题7：`st.markdown(html)` 将 HTML 转义为纯文本

**原因**：Streamlit 的 `st.markdown` 底层基于 Markdown-it 解析器，`<div class="...">` 不在白名单内被转义。

**解决**：改用 `st.html()`（Streamlit 1.38+），完全绕过 Markdown 解析器直接注入 DOM。

### 问题8：emoji 乱码问题

**问题**：部分终端或部署环境 emoji 显示为方框乱码。

**解决**：使用跨平台的 Unicode emoji（🏔️ 🏖️ 🏯 🥇 🥈 🥉），避免依赖系统特定字体。

### 问题9：部署版缺少悬停动效

**现象**：本地运行时按钮/Metric/卡片有放大效果，Streamlit Cloud 部署版没有。

**原因**：简化 CSS 修复 DOM 冲突时误删了 `:hover` 和 `transition` 样式。

**解决**：恢复到完整的悬停动效系统——按钮 `scale(1.04)` + 紫光阴影、Metric `translateY(-2px)` + 紫光阴影、旅行卡片 `translateY(-4px)` + 深影，暗色/浅色两种模式独立适配。

## 八、总结与展望

### 8.1 项目总结

本项目完整实现了一套"人脸检测→用户画像→情绪识别→旅游推荐"的端到端智能推荐系统，核心成果包括：

1. **零外部视觉依赖**：人脸检测/用户画像/情绪识别三大模块均基于纯 NumPy + Pillow，无需下载任何预训练模型
2. **跨平台兼容**：全部依赖均为纯 Python 包，任意 Python 版本、任意操作系统均可运行
3. **双模式推荐**：LLM 动态生成 + 内置模拟推荐自由切换，无 API Key 也能获得完整体验
4. **双主题 UI**：暗色/浅色模式自动适配，按钮/Metric/卡片均带悬停放大+发光阴影动效
5. **优雅降级**：每个视觉分析模块均内置兜底机制，确保系统在任何异常情况下都能正常运行

### 8.2 改进方向

1. **模型轻量化**：研究 ONNX Runtime 或 TFLite 推理，将小型 CNN 模型集成进纯 Python 管线而不增加编译依赖
2. **实时摄像头**：从静态照片分析扩展到 WebRTC 实时视频流的人脸检测与情绪追踪
3. **图片方案升级**：接入 Pexels/Unsplash 官方 API，为旅行卡片提供真实目的地照片
4. **多模态融合**：增加语音文字心情描述的情绪分析维度，与视觉情绪进行交叉验证
5. **用户系统**：接入数据库记录用户历史画像与偏好，实现真正的动态画像演化
