# TravelFace · 基于人脸画像与情绪识别的个性化旅游推荐系统

## 一、项目概述

### 1.1 项目背景

随着人工智能技术的快速发展，个性化推荐系统已广泛应用于电商、音乐、视频等领域。然而在旅游行业，传统的推荐方式仍主要依赖用户的历史浏览记录和评分数据，未能充分利用用户的即时情绪状态和视觉肖像信息。本项目创新性地将计算机视觉人脸检测、DeepFace 用户画像分析、情绪识别与 AIGC 智能体相结合，构建了一套"拍张照片即可获得专属旅行路线"的智能推荐系统。

### 1.2 核心功能

| 功能模块 | 描述 |
|---------|------|
| **人脸检测** | 基于 MediaPipe BlazeFace 神经网络模型，精准检测照片中人脸位置与数量 |
| **用户画像** | 基于 DeepFace 分析人脸年龄、性别，构建用户基础画像 |
| **情绪识别** | 识别人脸七种基本情绪（开心、悲伤、愤怒、惊讶、恐惧、厌恶、中性） |
| **旅游推荐** | 集成 OpenAI 兼容大语言模型 API，根据画像+情绪生成 3 条个性化路线 |
| **优雅降级** | 模型权重不可用时自动降级为估测值，确保系统在任何网络环境下正常运行 |

### 1.3 技术亮点

- **离线可用**：人脸检测使用 MediaPipe BlazeFace（模型随 pip 包分发），用户画像与情绪识别在权重未就绪时自动降级为估测值，无需网络即可完成完整分析流程
- **大模型双模式**：支持接入 OpenAI 兼容 API（默认豆包 doubao-1.5-pro-32k）进行智能推荐，未配置 API Key 时自动切换到内置模拟推荐数据
- **科技风 UI**：深色渐变背景 + 玻璃拟态卡片 + CSS 动效，按钮悬停放大、卡片悬浮上移、封面流光扫描等交互细节

## 四、模块详解

### 4.1 人脸检测模块 (FaceDetector)

**文件**：[src/vision/face_detector.py](file:///d:/Desktop/travelface/src/vision/face_detector.py)

**算法**：MediaPipe BlazeFace 短距离模型 (~224KB)

BlazeFace 是 Google 专为移动端设计的轻量人脸检测器，基于 MobileNetV2 骨干 + SSD 检测头，在单人正面照上近乎零误检。

**接口**：

```python
class FaceDetector:
    def __init__(self, min_confidence: float = 0.5)
    def detect(self, image: np.ndarray) -> list[dict]
```

**处理流程**：

```
BGR 图像 → cv2.cvtColor(BGR→RGB)
        → mp.Image(ImageFormat.SRGB)
        → vision.FaceDetector.create_from_options()
        → detector.detect()
        → 遍历 DetectionResult
        → 10% padding 扩展边界框
        → 返回 [{box, crop, detector}]
```

**参数说明**：

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `min_confidence` | 0.5 | 最小置信度阈值，低于此值的检测结果被丢弃 |
| `model_selection` | 1 | 0=短距离(<2m)，1=全距离(2-5m) |

**模型文件**：`assets/blaze_face_short_range.tflite`，首次运行时通过 `_get_model_path()` 自动定位。

### 4.2 用户画像模块 (UserProfiler)

**文件**：[src/vision/user_profiler.py](file:///d:/Desktop/travelface/src/vision/user_profiler.py)

**算法**：DeepFace 年龄/性别预测（多后端 fallback）

**优雅降级机制**：


**接口**：

**年龄段映射**：

| 预测年龄 | 年龄组 |
|---------|--------|
| < 18 | 少年 |
| 18-29 | 青年 |
| 30-44 | 中年 |
| ≥ 45 | 中老年 |

## 五、UI 设计

### 5.1 设计风格

采用**深色科技风 + 玻璃拟态**的设计语言：

- 背景：`linear-gradient(135deg, #0b0b2a, #1a1a4e, #16213e, #0f3460)` 四段深蓝紫渐变
- 卡片：`rgba(255,255,255,0.06)` 半透明 + `backdrop-filter: blur()` 毛玻璃效果
- 按钮：紫蓝渐变 `linear-gradient(135deg, #7b2ff7, #00d2ff)`

### 5.2 交互动效

| 元素 | 效果 | 实现 |
|------|------|------|
| 标题 | 渐变色 + 光晕呼吸 | `@keyframes titlePulse` |
| 按钮 | 悬停放大 1.05x + 发光阴影 | `transform: scale(1.05)` |
| Metric 卡片 | 悬停上移 2px + 紫光阴影 | `transform: translateY(-2px)` |
| 旅行卡片 | 悬停上移 4px | `transform: translateY(-4px)` |
| 封面图标 | 上下浮动 | `@keyframes iconFloat` |
| 封面光带 | 从左到右扫描 | `@keyframes shimmer` |

### 5.3 卡片封面设计

由于 LoremFlickr / Unsplash / AI 生成 API 在国内网络环境下均不可用，采用纯 CSS 封面方案：

- 每条路线使用不同的渐变色对（紫/红/蓝三组轮换）
- 大号场景图标（🏔️ 🏖️ 🏯）带浮动动画
- 白色粗体目的地名称（宽字距 + 文字阴影）
- 半透明光带从左到右无限循环扫描

### 5.4 HTML 渲染方案

初期使用 `st.markdown(html, unsafe_allow_html=True)`，发现 Markdown 解析器对 `<div class="travel-card">` 等自定义标签有白名单限制，导致 HTML 被转义成纯文本展示。最终改用 `st.html()`（Streamlit 1.38+）直出 DOM，彻底解决渲染问题。

## 七、开发问题与解决方案

### 问题1：OpenCV Haar Cascade 误检

**现象**：单人正面照被检出 2 张人脸（背景误判为人脸区域）。

**原因**：Haar Cascade 基于 2001 年的传统手工特征，缺乏神经网络级别的置信度评估机制。

**解决**：将人脸检测后端从 DeepFace 的 `opencv` 切换到 MediaPipe BlazeFace。BlazeFace 基于 MobileNetV2 + SSD，单人正面照近乎零误检，且模型随 `pip install mediapipe` 分发无需额外下载。

### 问题2：DeepFace 模型权重下载失败

**现象**：`age_model_weights.h5` 等文件从 GitHub 下载超时，导致 `RuntimeError: 画像分析所有后端均失败`。

**原因**：国内网络环境无法直连 GitHub Releases。

**解决**：在 `UserProfiler.__init__()` 和 `EmotionRecognizer.__init__()` 中加入权重文件预检 —— 启动时检查 `~/.deepface/weights/` 目录，文件不存在则设置 `_available = False`，后续调用直接返回估测值。避免每次请求都超时 30 秒。

### 问题3：`st.markdown(html, unsafe_allow_html=True)` 渲染为纯文本

**现象**：行程卡片 HTML 代码（`<div class="travel-card">`）以纯文本形式显示在页面上。

**原因**：Streamlit 的 `st.markdown` 底层基于 Markdown-it 解析器，对允许的 HTML 标签有严格白名单（`p/br/em/strong/a` 等），`<div class="...">` 不在白名单内被 `&lt;` 转义。

**解决**：改用 `st.html()`（Streamlit 1.38+ 引入的原生方法），完全绕过 Markdown 解析器，直接向 DOM 注入原始 HTML。

### 问题4：`uploaded_file.read()` 空字节

**现象**：`cv2.imdecode` 报 `!buf.empty()` 断言失败。

**原因**：`app.py` 中预览区调用一次 `read()` 后文件指针到末尾，分析区再次 `read()` 返回空字节。

**解决**：改用 `uploaded_file.getvalue()`（不移动指针），同时通过 `st.session_state["cached_image"]` 缓存解码后的 numpy 数组避免重复解码。

### 问题5：CSS 以纯文本形式显示在页面上

**现象**：`inject_custom_css()` 注入的 `<style>` 块中的 CSS 代码以纯文本形式显示。

**原因**：Python 三引号字符串每行前面 4 个缩进空格被 Markdown 解析器识别为代码块。

**解决**：使用 `textwrap.dedent()` 去掉公共前导空格，确保 `<style>` 标签顶格输出。

### 问题6：白底组件文字可读性差

**现象**：`p, span, div { color: #e8e8ff !important; }` 把 warning/info 框等白底组件内的文字也染成白色。

**解决**：将通用选择器替换为精准选择器 —— 主内容用 `[data-testid="stMarkdownContainer"]`，通知框用 `[data-testid="stNotification"]` 单独设为深色文字。

3. **双模式推荐**：LLM 动态生成 + 内置模拟推荐自由切换，无 API Key 也能获得完整体验
4. **科技风 UI**：深色渐变背景、玻璃拟态卡片、CSS 动效，视觉体验出色

### 8.2 改进方向

1. **模型本地化**：将 DeepFace 的年龄/性别/情绪模型文件打包进项目，彻底消除对 GitHub 网络依赖
2. **实时摄像头**：从静态照片分析扩展到 WebRTC 实时视频流的人脸检测与情绪追踪
3. **图片方案升级**：接入 Pexels/Unsplash 官方 API（非废弃接口），为旅行卡片提供真实目的地照片
4. **多模态融合**：增加语音文字心情描述的情绪分析维度，与视觉情绪进行交叉验证
5. **用户系统**：接入数据库记录用户历史画像与偏好，实现真正的动态画像演化
